简介

Docker Builder 是一个自动构建多架构 Docker 镜像的工具。它使用 GitHub Actions 在后台构建镜像，并推送到 Docker Hub。

功能特性

  • 多架构支持: amd64, arm64, arm
  • 一键构建: 简单的 Web 界面
  • 自动化: 使用 GitHub Actions
  • 实时日志: 构建过程中可查看进度

快速开始

1. 配置 Docker Hub

在 GitHub 仓库中配置 Secrets:

  • 访问仓库 Settings > Secrets and variables > Actions
  • 添加 DOCKERHUB_USER: 你的 Docker Hub 用户名
  • 添加 DOCKERHUB_TOKEN: Docker Hub Access Token

2. 获取 Docker Hub Token

  • 登录 Docker Hub
  • 进入 Account Settings > Security > New Access Token
  • 创建 Access Token 并复制

3. 使用 Web 界面

  • 访问 https://ac-all-sh.github.io/docker-builder/
  • 输入源镜像 (如 nginx)
  • 输入输出镜像 (如 yourusername/nginx)
  • 选择架构
  • 点击 "开始构建"

API 接口

GitHub Actions Workflow

可以通过 GitHub Actions 手动触发构建:

gh workflow run build.yml \
  -f image_source=nginx \
  -f image_name=yourusername/nginx \
  -f image_tag=latest \
  -f architectures=amd64,arm64

参数说明

参数 说明 必填
image_source 源镜像名称 是
image_name 输出镜像名称 是
image_tag 镜像标签 (默认: latest) 否
architectures 目标架构 (逗号分隔) 是

支持的架构

  • amd64 - x86_64
  • arm64 - 64位 ARM
  • arm - 32位 ARM (v7)

拉取镜像

# 拉取特定架构
docker pull --platform linux/amd64 yourusername/myapp:latest

# 拉取特定标签
docker pull yourusername/myapp:v1.0.0

Introduction

Docker Builder is a tool for automatically building multi-architecture Docker images. It uses GitHub Actions to build images in the background and push them to Docker Hub.

Features

  • Multi-architecture support: amd64, arm64, arm
  • One-click build: Simple web interface
  • Automation: Uses GitHub Actions
  • Real-time logs: View progress during build

Quick Start

1. Configure Docker Hub

Configure Secrets in your GitHub repository:

  • Go to Settings > Secrets and variables > Actions
  • Add DOCKERHUB_USER: Your Docker Hub username
  • Add DOCKERHUB_TOKEN: Docker Hub Access Token

2. Get Docker Hub Token

  • Login to Docker Hub
  • Go to Account Settings > Security > New Access Token
  • Create Access Token and copy it

3. Use Web Interface

  • Visit https://ac-all-sh.github.io/docker-builder/
  • Enter source image (e.g., nginx)
  • Enter output image (e.g., yourusername/nginx)
  • Select architectures
  • Click "Build Image"

API

GitHub Actions Workflow

Trigger builds manually via GitHub Actions:

gh workflow run build.yml \
  -f image_source=nginx \
  -f image_name=yourusername/nginx \
  -f image_tag=latest \
  -f architectures=amd64,arm64

Parameters

Parameter Description Required
image_source Source image name Yes
image_name Output image name Yes
image_tag Image tag (default: latest) No
architectures Target architectures (comma separated) Yes

Supported Architectures

  • amd64 - x86_64
  • arm64 - 64-bit ARM
  • arm - 32-bit ARM (v7)

Pull Images

# Pull specific architecture
docker pull --platform linux/amd64 yourusername/myapp:latest

# Pull specific tag
docker pull yourusername/myapp:v1.0.0
